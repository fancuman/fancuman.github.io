tips

docker-compose will create new network(named with folder name) so that containers can access each other (by using the format http://container_name).

use command “docker network ls” to show all the network

The frontend can use different path to access different backend api, for example:
/api/cars/, /api/food/, /api/weather/ …
nginx docker can forward different api access to different url by defining the rules(forward to different containers)

By this way, it only needs to expose 80 port for the frontend nginx docker container to the user. All the backend access can be reversed proxy through nginx and forwarded to the related backend container.
